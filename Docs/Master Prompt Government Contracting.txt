PROJECT NAME:
GovSignal AI


PROJECT TYPE:
AI-powered SaaS platform for federal contract discovery, monitoring, procurement intelligence, opportunity analysis, and proposal preparation.


==================================================
CORE MISSION
==================================================


Build a web-based platform that continuously monitors federal contract opportunities, intelligently matches them to businesses using AI, analyzes historical procurement behavior, and assists users in preparing proposal responses.


==================================================
IMPORTANT PRODUCT RULES
==================================================


- This is a WEBSITE, not desktop software.
- Do not build downloadable applications.
- Do not auto-submit government proposals without user approval.
- Human review is required before any submission.
- AI is an assistant, not a legal authority.
- AI must never invent company qualifications or certifications.
- AI must clearly label uncertainty and missing information.
- AI must only use verified user/company data.


==================================================
POSITIONING
==================================================


Do NOT market this as:
“AI automatically applies to contracts.”


Market this as:
“AI-powered federal contract intelligence and proposal preparation.”


MVP TAGLINE:
Find better federal contracts. Understand them faster. Draft proposals in minutes.


==================================================
CORE USER FLOW
==================================================


1. User creates account.
2. User creates company profile.
3. User uploads company capabilities and documents.
4. User selects:
   - NAICS codes
   - PSC codes
   - keywords
   - agencies
   - exclusions
5. System continuously monitors SAM.gov.
6. AI analyzes opportunities.
7. AI scores opportunity fit.
8. AI analyzes historical award data.
9. AI alerts users to high-value matches.
10. AI generates proposal drafts and compliance checklists.
11. User reviews and exports proposal package.
12. User manually submits or follows guided submission instructions.

==================================================
WATCHLIST SYSTEM
==================================================


GOAL:
Allow users to continuously monitor opportunities using:
- keywords
- semantic AI concepts
- NAICS codes
- PSC codes
- agencies
- set-asides
- location preferences
- budget ranges
- exclusions


WATCHLIST EXAMPLES:
- AI automation
- XR training
- Unity development
- Cybersecurity
- Mobile app modernization
- Simulation systems


NEGATIVE FILTERS:
- Construction
- Janitorial
- Staffing
- On-site only
- Clearance required


WATCHLIST WORKFLOW:
1. User creates watchlist.
2. System stores preferences.
3. Scheduled jobs run hourly or daily.
4. System pulls newly posted opportunities.
5. AI expands semantic keywords.
6. AI scores opportunity relevance.
7. Matches are saved.
8. User receives notifications.


==================================================
SEMANTIC MATCHING
==================================================


The system must not rely only on exact keywords.


Example:
“virtual reality training”


Should also match:
- XR simulation
- immersive environments
- mixed reality systems
- interactive training systems
- simulation platforms


==================================================
AI RELEVANCE SCORE
==================================================


Score opportunities from 0-100 using:
- NAICS match
- PSC match
- semantic similarity
- keyword overlap
- certifications
- agency preference
- location preference
- budget fit
- capability alignment
- past performance similarity


MATCH SCORE OUTPUT:
- Score
- Confidence level
- Reasons to bid
- Risks
- Missing information
- Bid/no-bid recommendation


HIGH PRIORITY RULE:
If score > 85:
Mark as HIGH PRIORITY.

==================================================
AGENCY SPENDING ANALYSIS
==================================================


Track:
- total agency spend by NAICS
- total agency spend by PSC
- recurring procurement categories
- annual spending trends
- seasonal purchasing behavior


==================================================
PRICE BENCHMARKING
==================================================


Estimate:
- average award size
- typical contract range
- historical pricing patterns
- likely expected budget


OUTPUT EXAMPLES:
- “Average historical award: $420,000”
- “Agency spent $8.2M on XR simulation over 3 years”
- “Typical contracts range between $150K-$600K”


==================================================
INCUMBENT ANALYSIS
==================================================


Track:
- recurring winners
- incumbent contract holders
- contractor dominance patterns
- repeat agency relationships


OUTPUT EXAMPLES:
- “Incumbent contractor won 4 consecutive awards”
- “Agency frequently awards to small businesses”
- “Incumbent retention rate: 62%”


==================================================
REBID FORECASTING
==================================================


GOAL:
Estimate likely rebid timing using historical awards.


OUTPUT:
- “Likely recompete expected within 45 days”
- “Recurring annual procurement detected”


IMPORTANT:
Historical intelligence is a supporting signal, not a guaranteed predictor.

==================================================
FALLBACK RULES FOR AI HALLUCINATIONS
==================================================


If AI invents APIs:
Return to official SAM.gov documentation.


If AI confuses awards with opportunities:
Correct immediately.


If AI skips database schema:
Force schema creation before coding.


If AI produces vague code:
Request file-by-file implementation.


If AI attempts to build everything simultaneously:
Return to current MVP phase only.


If AI invents company qualifications:
Replace with:
“MISSING USER DATA”


If AI suggests scraping before official APIs:
Use official APIs first.


If AI writes freeform automation logic:
Convert outputs to structured JSON.
